Using the API


You can use this API for testing your modules, without using e.g. POSTMAN for sending HTTP-Requests. The 'module_id' and 'resource_id' are usually a part of the URL. 'request_method' is the HTTP-Method, GET, POST, PUT, DELETE are in use till now but the method is forwarded to the adressed module, which can implement further methods. The 'json_data' is optional and depends only on the adressed module.

Example: Installing the api_games-module

{
"module_id": "INSTALLER",
"resource_id": "install",
"request_method": "PUT",
"json_data":
{
"module_id" : "api_games"
}
}
